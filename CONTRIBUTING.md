# Contributing to Shelby Stream

We welcome contributions to the Shelby Stream decentralized media marketplace!

## 🛠 Development Workflow

1.  **Fork the Repository**: Create a fork of the `shelby-stream` repository.
2.  **Clone the Repository**: Clone your fork to your local machine.
3.  **Install Dependencies**: Run `npm install` to install all required packages.
4.  **Create a Branch**: Create a new branch for your feature or bug fix.
5.  **Make Changes**: Implement your changes and ensure they follow the project's coding style.
6.  **Test Your Changes**: Run the development server and verify your changes.
7.  **Submit a Pull Request**: Submit a pull request to the main repository.

## 📜 Coding Style

- Use **TypeScript** for all frontend and backend code.
- Follow **Tailwind CSS** utility-first principles for styling.
- Use **functional components** and **hooks** in React.
- Ensure all new components are **responsive** and **accessible**.

## 🚀 Smart Contract Development

- The marketplace logic is written in **Move**.
- Ensure all new Move functions are properly documented and tested.
- Follow the **Aptos Framework** best practices for security and efficiency.
